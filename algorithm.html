CheatSheet

Первые шаги после получения таски

- Убедиться, что задание понятно и корректно

- Опционально можно прощёлкать тест руками

- Сравнить запросы и ответы между Сваггером(ОР) и Постманом(ФР)

- Продумать основную концепцию будущих тест-кейсов: позитивные и негативные(если в доке есть инфа по 4ХХ ответам)

- Приступая делать таску - переносим её в таск-борде в статус "in progress"

- !Важно: Создавать ветку для новой задачи через консоль командой "git checkout -b ..." - только с master ветки


Работа в IDE

- Стандартный флоу создания классов: Client, Contract, Entity, Cest

- !Важно: если для кейсов или аналогичных тестов используются одинаковые private функции - создаём Helper/Builder

- После написания теста пробегись по нему взгядом "со стороны": нейминги всех функций/переменных будут понятны любому ТА?

- #Совет: После написания теста можно пройтись глазами по своему коду и задать себе вопросы: Я написал хороший для чтения/поддержания код? Что я могу сейчас улучшить в нем?

- Перед коммитом не забыть прогнать линтером

- Если во время выполнения таски были мержи других ТА, то можно также актуализировать ветку


Перед ПР

- Если таска связана с написанием нового теста: в Description пишем кейсы(актуальные шаги из написанного теста)

- Если таска связана с фиксом теста: в Description уже должно быть описание проблемы, а также ОР и ФР результаты. Если их нет - пишем. Можно указать там же, что данный баг пофикшен
